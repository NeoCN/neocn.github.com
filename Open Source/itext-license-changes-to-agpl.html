iText License 更换为 AGPL

Dec 8th, 2009 | Comments

著名的Java PDF类库iText于12月7日发布了5.0版，新版本除了更改了包名、要求JDK5.0、修复Bug外，最主要的一个改变是其License的改变，许可由MPL/LGPL更换为AGPL。 

Changes made by Bruno : Changed the license from MPL/LGPL to AGPL
对于GPL、LGPL是比较常见的，这个AGPL到底有什么约束呢？我们一般情况下会把iText作为类库来使用。

GPL许可要求任何使用GPL代码的软件也必须在GPL许可下发布，提供源代码；LGPL则允许在只对LGPL许可的代码做静态引用，即作为类库调用的情况下，不开源代码；而AGPL，其实是对GPL的一个补充，因为GPL许可下的代码和软件，在用作提供服务的情况下，比如用于提供Web Service服务，是不受GPL许可约束的，AGPL即是针对这种情况增加了约束条款。

以下内容摘录自网络： 

AGPL这个协议的制定是为了避免一个GPL/LGPL协议中的漏洞，称之为 Web Service Loopwhole。
这主要是由于 GPL是针对传统的软件分发模式的商业模式(以微软为代表), 如果你使用的GPL的代码作为
基础完成你自己的软件，如果你要分发你的软件，你的软件必须也是GPL的。随着以Google为代表的软件
作为服务的互联网公司的兴起，它们的“不分发软件，为客户提供网络服务”的商业模式就不受GPL协议的
约束，所以Google公司在构筑他的搜索引擎的时候可以随心所欲的拿现有的GPL协议的开源代码，无需开
源他的修改成果。AGPL协议在GPL协议的基础上加上了这个约束。
所以，iText新的开源许可，对于商业应用来讲，比GPL许可更不友好。
